Architecture overview
How it works:
Website → sends command → your Node.js server → stores in queue → Studio plugin polls every 2s → executes → sends result back → server → website shows result.

Files you need:
dashboard.html ← this file (website)
server.js ← Node.js backend
studio-plugin.lua ← Roblox Studio plugin Download all three from the dashboard. Host server.js on Railway, Render, or any Node host. Load the plugin into Studio.
Backend endpoints used
POST /api/command — website sends command + token
GET /api/poll/:token — plugin polls for pending commands
POST /api/result — plugin posts execution result
GET /api/result/:token — website checks for result
POST /api/verify — verifies Roblox username
